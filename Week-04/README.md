# Week 04: Final Project Week - Machine Learning Application

## Overview
This is the **FINAL INTENSIVE PROJECT WEEK** for your TVET internship. You will design, implement, and document a complete machine learning application from scratch.

## 🎯 Project Selection (CHOOSE ONE)

You **MUST** choose ONE of the following two projects:

### Project Option 1: Weather Data & Farmer Loss Prediction System (Rwanda-focused)
Build a data-driven application that predicts potential farmer losses using:
- Rainfall data
- Month/season
- Price of manure at planting time
- **Uses Machine Learning with Linear Regression**
- **Simulated/sample datasets ONLY (no real farmer data)**

**Outputs:**
- Predicted loss or risk level
- Explanation of factors influencing the prediction

---

### Project Option 2: Informal Economy Credit Scoring Simulator
Build a simulator that estimates a credit score for informal workers using:
- Income consistency
- Transaction frequency
- Savings behavior
- Risk flags
- **Simulation only (no real money, no real users)**

**Outputs:**
- Credit score (0–100 or categories)
- Loan eligibility decision
- Explanation of scoring logic

---

## 📋 Requirements for BOTH Projects

### Must Include:
✅ Clear problem statement  
✅ Data modeling explanation  
✅ Algorithm explanation (step-by-step, in comments or README)  
✅ Edge case handling  
✅ Final reflection  

### Technical Requirements:
- **Tech Stack:** Python (preferred) or Web (HTML/CSS/JS + Python logic)
- **Machine Learning:** Simple and explainable (Linear Regression only)
- **Focus:** LOGIC over UI beauty
- **Ethics:** No real betting, no real money, no real personal data

## 🧩 Weekly LeetCode Challenge

**Problem (Easy):** [FizzBuzz](https://leetcode.com/problems/fizz-buzz/)

> Given an integer `n`, return a string array where for each `i` from `1` to `n`: output `"FizzBuzz"` if divisible by 3 and 5, `"Fizz"` if divisible by 3, `"Buzz"` if divisible by 5, or the number itself as a string.

This classic problem builds conditional logic and iteration skills — the same structured thinking required for designing ML decision rules.
