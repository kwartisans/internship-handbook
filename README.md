# internship-handbook

Welcome to the **Kigali Web Artisans Software Engineering Internship Program**! This is a comprehensive 4-week, 20-day remote internship designed for high school students who want to gain hands-on experience in software engineering, culminating in a complete machine learning application.

---

## 🎯 Program Overview

This intensive program transforms you from a beginner to someone capable of building a complete machine learning application. You'll learn:
- **Version control** and professional development workflows
- **Backend development** with PHP, Node.js, and databases
- **Machine learning** fundamentals and applications
- **Web development** with HTML, CSS, and JavaScript
- **Real-world project delivery** from concept to deployment

---

## 📅 Complete Internship Roadmap

### 🔰 [Week 01: Fundamentals & Development Environment](Week-01/README.md) (Days 01-05)

**Theme**: Building Your Engineering Foundation

**What You'll Learn:**
- Master Git and GitHub for version control and remote collaboration
- Set up professional development environment (VS Code, terminal, etc.)
- Build practical backend skills with PHP, databases (SQL/NoSQL), and Node.js
- Develop algorithmic problem-solving abilities through daily LeetCode challenges
- Learn professional code documentation and communication practices

**Week 1 Project Arc:**
By Friday, you'll have built a complete data pipeline:
- **Day 1**: Git/GitHub mastery and environment setup

- **Day 4**: Node.js/Express API exposing your data
- **Day 5**: Code review, refactoring, and weekly retrospective

**Why This Matters**: Week 1 creates the technical foundation for everything ahead. The data pipeline you build mirrors real-world engineering and prepares you for ML data handling.

### 🚀 [Week 04: Final Project - Machine Learning Application](Week-04/README.md) (Days 16-20)

**Theme**: Complete ML Project from Concept to Delivery

**Your Mission**: Design, implement, and document a complete machine learning application from scratch.

**Choose ONE Project:**

**Option 1: Weather Data & Farmer Loss Prediction System** (Rwanda-focused)
- Predict potential farmer losses using rainfall, season, and manure price data
- Uses Linear Regression for predictions
- Output: Predicted loss/risk level with explanatory factors

**Option 2: Informal Economy Credit Scoring Simulator**
- Estimate credit scores for informal workers
- Input: Income consistency, transaction frequency, savings, risk flags
- Output: Credit score (0-100), loan eligibility, scoring explanation

**Week 4 Structure:**
- **Day 16**: Project planning and design (problem statement, data model, algorithm design)
- **Day 17**: Data preparation and algorithm setup
- **Day 18**: Core implementation and ML model building
- **Day 19**: Testing, edge case handling, and refinement
- **Day 20**: Documentation and final presentation

**Project Requirements:**
- Clear problem statement and algorithm explanation
- Ethical implementation (simulated data only, no real betting/money/personal data)
- Edge case handling and error management
- Comprehensive documentation
- Ability to explain and defend every line of code

**Evaluation Criteria:**
- Problem Understanding (20%)
- Algorithm Implementation (30%)
- Code Quality & Documentation (20%)
- Edge Case Handling (15%)
- Oral Defense & Explanation (15%)

**Why This Matters**: This is your capstone project—demonstrating everything you've learned. It's a portfolio piece showcasing your ability to deliver a complete, production-ready application.

## 🎓 Learning Philosophy

This internship follows a **progressive mastery model**:

1. **Week 1**: **Foundation** - Core skills and tools (Git, backend, databases)
2. **Week 2**: **Exploration** - Exposure to ML, mobile, and professional practices
3. **Week 3**: **Creation** - Building interactive applications with web technologies
4. **Week 4**: **Integration** - Combining all skills into a complete ML project

**Key Principles:**
- **Hands-on Learning**: Code every day, learn by doing
- **Real-World Tools**: Use technologies that professional engineers use
- **Progressive Complexity**: Each week builds on previous knowledge
- **Project-Based**: Culminate learning in tangible deliverables
- **Remote Collaboration**: Master distributed team workflows
- **Professional Habits**: Documentation, code review, and communication

## 📋 Setup Script

We've included a Bash script to set up the directory structure. If you need to recreate or set up the structure elsewhere, you can use:

```bash
bash setup-internship-structure.sh
```

The script will:
1. Create a main `internship-handbook` directory (or use the current directory if already inside it)
2. Create Week-01 through Week-04 folders
3. Create Day folders with sequential numbering (Day-01 to Day-20)
4. Add a README.md file to each day folder
